Assigned Classification Model – Basic Concepts

  1. What the model is (Nearest Centroid)

    Nearest Centroid is a very simple supervised classification algorithm. The main idea is that each class has a centroid, which is basically the average of all feature values in that class. When a new data point comes in, the algorithm assigns it to the class whose centroid is closest. This method is intuitive and fast, especially when classes form well-defined groups in the data.

  2. How it works in general terms

    First, the algorithm calculates the centroid for each class using all the training data. Then, for any new observation, it measures the distance to each centroid and assigns the observation to the nearest class. The most common distance metrics are Euclidean or Manhattan, depending on the data. The advantage is that it doesn’t require complex or iterative training, making it very fast and easy to use.

  3. Main characteristics

    • Simple and very fast to train and predict.
    • Works best when classes are clearly separated and roughly spherical in the feature space.
    • Non-parametric: it does not assume any specific data distribution.
    • Can handle multi-class problems naturally.
    • Consumes low memory and computational resources, even for medium-sized datasets.

  4. Advantages and limitations

    • Advantages: fast, easy to understand, few parameters to tune, low memory usage, suitable for small and medium datasets.
    • Limitations: sensitive to outliers, can perform poorly if classes overlap or have irregular shapes, less effective for high-dimensional sparse data without feature scaling.

  5. Type of classification problems where it can be applied

    Nearest Centroid is useful when classes are compact and well-separated in the data. It works for both binary and multi-class classification tasks, especially when each feature meaningfully contributes to separating classes. Typical applications include document classification, simple image recognition, or any situation where averaging each class’s data gives a meaningful representation.

  6. Real-world use example

    One practical example is document classification, where each document is represented as a vector of word frequencies or embeddings. The algorithm assigns a document to the category whose centroid of word profiles is closest. Another example is preliminary medical diagnosis, where patients are grouped based on key measurements like blood tests or vital signs, allowing for fast and interpretable classification.
